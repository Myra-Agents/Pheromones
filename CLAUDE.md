# @myra/shared ‚Äî Claude Code Instructions

Shared TypeScript types, API contracts, and pure domain helpers for the Myra
Agents ecosystem. **No runtime, no secrets, no side effects** ‚Äî types and pure
functions only. Consumed as the `packages/shared` git submodule by **app** and
**hub**, and mirrored (by hand) by the Rust **server**.

> **Workspace code index:** when this clone lives inside the Myrastack dev
> workspace, `../.claude/index/shared.md` (gitignored, regenerated by `../index.sh`)
> is a greppable index of this repo ‚Äî `## dirs` / `## files` / `## symbols`
> (`path:line:definition`). Grep it to locate code instead of scanning the
> tree; if stale, run `(cd .. && ./index.sh shared)`.

Part of the [Myra-Agents](https://github.com/orgs/Myra-Agents/repositories) org:
app (public, desktop) ¬∑ **shared (public, this)** ¬∑ hub (private, SaaS) ¬∑
server (private, Rust sidecar) ¬∑ plugins (public).

## What this is / is not

- **Is:** the single source of truth for the data models + the client‚áÑbackend
  command contract. Pure, dependency-free, not published to npm.
- **Is not:** a buildable package. There is no build step and no test project ‚Äî
  consumers import the `.ts` directly via the workspace + the `exports` map in
  `package.json`. Don't add bundling, a `dist/`, or runtime deps.

## Layout (`src/`)

- `types/` ‚Äî `kanban.ts`, `schedule.ts`, `settings.ts`: the data models. These
  **mirror the Rust structs** in the server (`models.rs`) and the app's
  `src-tauri` ‚Äî camelCase field names must match byte-for-byte.
- `contract.ts` ‚Äî the client‚áÑbackend command contract + route table.
- `hub-contract.ts` ‚Äî the desktop‚áÑhub API contract.
- `domain/` ‚Äî pure helpers: `cards`, `schedules`, `settings`, `agent`,
  `command`, `ids`. No I/O, no `Date.now()` non-determinism leaking into outputs
  where it can be passed in.
- `store.ts`, `dispatch.ts` ‚Äî shared store + dispatch utilities.
- `index.ts` ‚Äî the public barrel (the `.` export).

## Conventions

- **Changing a model is a cross-repo change.** A field added here must be added
  to: the Rust struct in `server/src/models.rs`, the app's `src/types/` + every
  Rust struct literal, and the hub if it touches the hub contract. camelCase via
  `#[serde(rename_all = "camelCase")]` on the Rust side.
- Keep everything tree-shakeable and side-effect-free; the `exports` map is the
  API surface ‚Äî update it when you add a new entry point.
- After editing, consumers pick it up by bumping their submodule pointer
  (`git submodule update --remote packages/shared`), not via npm.

## Verify

```bash
npx tsc --noEmit      # the only gate ‚Äî types must compile standalone
```

## Branching

GitFlow-lite, **org-wide** across all Myra-Agents repos:

- `main` ‚Äî stable, released code; **tagged releases only**, never commit straight to it.
- `develop` ‚Äî **default branch**; all day-to-day work integrates here.
- `feature/<slug>` ¬∑ `fix/<slug>` ¬∑ `chore/<slug>` ‚Äî short-lived, branch off `develop`, PR back into `develop`.
- Release: merge `develop` ‚Üí `main` + tag (`vX.Y.Z`; server uses `server-vX.Y.Z`).
- Hotfix: branch off `main`, PR into `main`, then merge `main` back to `develop`.

Open PRs against `develop`. Conventional Commit subjects. One logical change per PR.
